# SaberLight

SaberLight is a high-fidelity, interactive lightsaber system designed for responsive two-player combat. By leveraging **capacitive sensing**, the system accurately distinguishes between defensive parries (blade-to-blade) and offensive scoring (body hits) in real-time.

## System Overview
The project integrates embedded firmware, wireless synchronization, and custom PCB design to deliver a seamless dueling experience with real-time feedback.

### Key Features
* **Capacitive Hit Detection:** Advanced sensing logic to differentiate contact types without mechanical switches.
* **Wireless Synchronization:** Real-time point and health tracking between two sabers.
* **Integrated UI:** Hilt-mounted LCDs for health monitoring and system status.
* **Audiovisual Feedback:** Low-latency LED patterns and audio responses synchronized to combat events.

## Repository Structure

### 📂 Hardware (`/hw`)
Contains the full hardware design suite, including:
* **Schematics & Layouts:** KiCad board files optimized for hilt integration and power management.
* **System Design:** Documentation for the TM4C-based architecture and capacitive touch interface.

### 📂 Software (`/sw`)
The core firmware stack for the dueling system:
* **Application Logic:** Real-time hit detection, state machine management, and point synchronization.
* **Drivers:** Low-level firmware drivers for the **ARM Cortex-M4 (TM4C)** microcontroller, including LCD interfacing and wireless communication protocols.
* **Feedback Engine:** Processing for LED patterns and synchronized sound effects.

### 📂 Resources (`/Resources`)
Technical documentation and reference materials essential for the system’s development:
* **TM4C Datasheets:** Detailed specifications for the MCU architecture.
* **System Design Guidelines:** Standards for signal integrity and power distribution in embedded hardware.
